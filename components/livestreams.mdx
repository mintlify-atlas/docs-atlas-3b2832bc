# LiveStreams Component

The LiveStreams component displays live and recorded event streams with real-time status tracking, countdown timers, and embedded video players. It supports multiple streaming platforms and automatically updates stream status based on date/time.

## Overview

The LiveStreams component provides:
- **Stream cards** - Visual cards for each livestream with status badges
- **Status tracking** - Automatically determines if stream is upcoming, live, or recorded
- **Countdown timers** - Live countdown for upcoming streams
- **Modal viewer** - Full-screen modal with embedded video player
- **Platform support** - YouTube, Facebook, Twitch, and custom embeds

## Props

The LiveStreams component does not accept any props. It's a self-contained section component.

## Key Features

### 1. Dynamic Status Tracking

Streams automatically update status based on current date/time:

```typescript
const visibleStreams = STREAMS.map(getStreamWithStatus).filter((stream) => {
  const now = new Date();
  const streamEndDate = new Date(`${stream.date}T${stream.endTime}:00`);

  // Hide streams that ended without recording
  if (now > streamEndDate && !stream.videoId) {
    return false;
  }

  return true;
});
```

Status is determined by the `getStreamWithStatus` helper from `../utils/streamHelpers`.

### 2. Countdown Timer Component

Nested component displays live countdown for upcoming streams:

```typescript
const CountdownTimer = ({ timeUntilStart }: { timeUntilStart: number }) => {
  const [countdown, setCountdown] = useState(formatCountdown(timeUntilStart));

  useEffect(() => {
    const interval = setInterval(() => {
      const now = Date.now();
      const remaining = timeUntilStart - (Date.now() - now);
      if (remaining > 0) {
        setCountdown(formatCountdown(remaining));
      }
    }, 1000);

    return () => clearInterval(interval);
  }, [timeUntilStart]);

  return (
    <div className={styles.countdown}>
      <div className={styles.countdownLabel}>Comienza en:</div>
      <div className={styles.countdownTime}>
        {countdown.hours > 0 && (
          <div className={styles.timeUnit}>
            <span className={styles.timeValue}>
              {String(countdown.hours).padStart(2, "0")}
            </span>
            <span className={styles.timeLabel}>hrs</span>
          </div>
        )}
        {/* minutes and seconds */}
      </div>
    </div>
  );
};
```

### 5. Keyboard and Accessibility Support

Modal supports keyboard navigation:

```typescript
useEffect(() => {
  const handleEscape = (e: KeyboardEvent) => {
    if (e.key === "Escape" && isModalOpen) {
      closeModal();
    }
  };

  if (isModalOpen) {
    document.addEventListener("keydown", handleEscape);
    document.body.style.overflow = "hidden";

    const firstFocusable = modalRef.current?.querySelector("button");
    (firstFocusable as HTMLElement)?.focus();
  } else {
    document.body.style.overflow = "";
  }

  return () => {
    document.removeEventListener("keydown", handleEscape);
    document.body.style.overflow = "";
  };
}, [isModalOpen]);
```

## Stream Status Logic

Status is determined by comparing current time with stream dates:

- **upcoming/scheduled**: Current time < start time
- **live**: Current time between start and end time
- **recorded**: Current time > end time AND has `videoId`
- **Hidden**: Current time > end time AND no `videoId`

## Accessibility Features

- **ARIA roles**: Modal uses `role="dialog"` and `aria-modal="true"`
- **ARIA labels**: Modal title uses `aria-labelledby`
- **Keyboard support**: ESC key closes modal
- **Focus management**: Focus moves to first interactive element in modal
- **Body scroll lock**: Prevents background scrolling when modal open

## Performance Considerations

- **Conditional rendering**: Streams without recordings are hidden after end time
- **Cleanup**: Event listeners and intervals are properly cleaned up
- **Lazy loading**: Video embeds only load when modal opens
