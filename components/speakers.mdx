# Speakers Component

The Speakers component displays all event speakers with filtering capabilities by day and search term. Each speaker card shows their information, topic, and schedule.

## Overview

The Speakers component provides:
- **Speaker cards** - Grid display of all speakers with bio and details
- **Search functionality** - Real-time search by name, topic, or role
- **Day filters** - Filter speakers by event day
- **Animations** - Fade-in and hover lift effects

## Props

The Speakers component does not accept any props. It's a self-contained section component.

## Key Features

### 1. Search Functionality

Real-time search filters speakers by name, topic, or role:

```tsx
<div className={styles.searchBox}>
  <Search className={styles.searchIcon} size={20} />
  <input
    type="text"
    placeholder="Buscar por nombre, tema o rol..."
    value={searchTerm}
    onChange={(e) => setSearchTerm(e.target.value)}
    className={styles.searchInput}
    aria-label="Buscar ponentes"
  />
</div>
```

Search implementation:

```typescript
const filteredSpeakers = SPEAKERS_DATA.filter((speaker) => {
  const matchesDay = selectedDay === "Todos" || speaker.day === selectedDay;
  const matchesSearch =
    speaker.name.toLowerCase().includes(searchTerm.toLowerCase()) ||
    speaker.topic.toLowerCase().includes(searchTerm.toLowerCase()) ||
    speaker.role.toLowerCase().includes(searchTerm.toLowerCase());
  return matchesDay && matchesSearch;
});
```

### 2. Day Filters

Filter speakers by event day:

```tsx
const DAYS = ["Todos", "Miércoles 10", "Jueves 11", "Viernes 12"];

<div className={styles.filters}>
  {DAYS.map((day) => (
    <button
      key={day}
      onClick={() => setSelectedDay(day)}
      className={`${styles.filterBtn} ${
        selectedDay === day ? styles.active : ""
      }`}
      aria-pressed={selectedDay === day}
    >
      {day}
    </button>
  ))}
</div>
```

### 3. Speaker Cards

Each speaker is displayed in a card with all details:

```tsx
<div key={speaker.id} className={styles.speakerCard}>
  <div className={styles.imageContainer}>
    <div className={styles.imagePlaceholder}>
      {speaker.name
        .split(" ")
        .map((n) => n[0])
        .join("")}
    </div>
  </div>
  <div className={styles.content}>
    <h3 className={styles.name}>{speaker.name}</h3>
    <p className={styles.role}>{speaker.role}</p>
    <p className={styles.bio}>{speaker.bio}</p>
    <div className={styles.topic}>
      <span className={styles.topicLabel}>Tema:</span>
      <span className={styles.topicName}>{speaker.topic}</span>
    </div>
    <div className={styles.schedule}>
      <Calendar size={16} />
      <span>
        {speaker.day} - {speaker.time}
      </span>
    </div>
  </div>
</div>
```

**Image Placeholder**: Since speaker images use placeholders, the component generates initials from the speaker's name as a fallback.

### 4. No Results Message

```tsx
{filteredSpeakers.length === 0 && (
  <div className={styles.noResults}>
    <p>No se encontraron ponentes con los criterios seleccionados</p>
  </div>
)}
```

### 5. Animations

```typescript
useEffect(() => {
  if (sectionRef.current) {
    fadeInUp(sectionRef.current, 1000);
  }

  const timer = setTimeout(() => {
    batchHoverLift(`.${styles.speakerCard}`);
  }, 500);

  return () => clearTimeout(timer);
}, []);
```

- **fadeInUp**: Section fades in from bottom on mount
- **batchHoverLift**: Speaker cards get hover lift effect

## Customization

### Adding New Speakers

Add new speaker objects to the `SPEAKERS_DATA` array:

```typescript
const SPEAKERS_DATA: Speaker[] = [
  // ... existing speakers
  {
    id: 7,
    name: "New Speaker Name",
    role: "Professional Title",
    bio: "Biography and expertise...",
    topic: "Presentation Topic",
    day: "Miércoles 10",
    time: "14:00 - 15:00",
    image: "/path-to-image.jpg",
  },
];
```

### Updating Day Filters

Modify the `DAYS` constant to match your event schedule:

```typescript
const DAYS = ["Todos", "Day 1", "Day 2", "Day 3"];
```

## Accessibility

- **Section ID**: `id="ponentes"` for anchor navigation
- **ARIA labels**: Search input has descriptive `aria-label`
- **ARIA pressed**: Filter buttons use `aria-pressed` for state
- **Semantic HTML**: Proper heading hierarchy and list structure

## Performance Considerations

- **Filtered rendering**: Only renders speakers matching current filters
- **Memoization opportunity**: Consider using `useMemo` for `filteredSpeakers` if the data set grows large
- **Lazy loading**: Images could be lazy-loaded for better performance
