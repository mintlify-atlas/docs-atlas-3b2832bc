## Favicon Setup

Favicons are referenced in `index.html:12-14`:

```html
<!-- Favicons -->
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png" />
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png" />
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png" />
```

### Creating Favicons

1. Create or design your favicon (recommended: 512x512px PNG)
2. Use a favicon generator tool (e.g., [favicon.io](https://favicon.io), [RealFaviconGenerator](https://realfavicongenerator.net/))
3. Generate multiple sizes:
   - `favicon-16x16.png`
   - `favicon-32x32.png`
   - `apple-touch-icon.png` (180x180px)
   - `favicon.ico` (optional, for older browsers)
4. Place all files in the `/public` folder
5. Update `index.html` if using different filenames

### Manifest File

The PWA manifest is at `/public/manifest.json` and referenced at `index.html:17`:

```html
<link rel="manifest" href="/manifest.json" />
```

Edit `manifest.json` to update app name, colors, and icons:

```json
{
  "short_name": "DEV WEEK",
  "name": "DEV WEEK 2025 - Semana del Desarrollo",
  "icons": [
    {
      "src": "favicon-32x32.png",
      "sizes": "32x32",
      "type": "image/png"
    },
    {
      "src": "logo-192x192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "logo-512x512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ],
  "start_url": "/",
  "display": "standalone",
  "theme_color": "#61398C",
  "background_color": "#ffffff"
}
```

Make sure to create 192x192 and 512x512 versions of your logo for PWA support.

1. Design a 1200x630px image with:

   - Brand colors
2. Save as `og-image.jpg` in `/public`
3. Update the full URL in `index.html` (lines 54 and 68)

## Adding New Images

### Step 1: Add to Public Folder

```bash
cp your-new-image.png /public/
```

### Step 2: Reference in Code

In a React component:

```tsx
<img 
  src="/your-new-image.png" 
  alt="Descriptive alt text"
  className={styles.yourClass}
/>
```

In CSS:

```css
.background {
  background-image: url('/your-new-image.png');
}
```

### Step 3: Optimize Images

For best performance:

- Use **WebP** format for modern browsers (with PNG/JPG fallback)
- Compress images (tools: [TinyPNG](https://tinypng.com/), [Squoosh](https://squoosh.app/))
- Use appropriate dimensions (don't serve 4000px images if you only display 400px)
- Consider lazy loading for below-the-fold images

   }

   - Square aspect ratio (1:1)
   - At least 400x400px
   - Professional headshot
   - Consistent style across all speakers

## robots.txt and sitemap.xml

Two important SEO files are in `/public`:

- **`robots.txt`**: Controls search engine crawling
- **`sitemap.xml`**: Lists all pages for search engines

Update `robots.txt` with your sitemap URL:

```txt
User-agent: *
Allow: /

Sitemap: https://semanainformatica2025.expandeya.com/sitemap.xml
```

Update `sitemap.xml` with your actual domain and pages:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://semanainformatica2025.expandeya.com/</loc>
    <lastmod>2025-12-01</lastmod>
    <priority>1.0</priority>
  </url>
</urlset>
```

## Asset Organization Best Practices

1. **Use subfolders** for organization:
   ```
   /public/
   ├── logos/
   ├── speakers/
   ├── backgrounds/
   └── icons/
   ```

2. **Use descriptive names**: `event-logo-2025.png` instead of `logo1.png`

3. **Keep originals**: Store high-res originals outside `/public`, only put optimized versions in `/public`

4. **Document usage**: Comment in code where each asset is used

5. **Version control**: Use Git LFS for large images if needed
