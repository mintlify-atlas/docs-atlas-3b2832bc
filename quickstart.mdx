## Get started in 5 minutes

Follow these steps to clone the repository, install dependencies, and run the development server.

<Steps>
  <Step title="Clone the repository">
    Clone the DEV WEEK 2025 source code to your local machine:

    ```bash
    git clone <repository-url>
    cd vite-react-typescript-starter
    ```

    <Note>
      Replace `<repository-url>` with the actual Git repository URL for your project.
    </Note>
  </Step>

  <Step title="Install dependencies">
    Install all required packages using npm:

    ```bash
    npm install
    ```

    This will install:
    - React 18.3.1 and React DOM
    - Vite 5.4.2 build tool
    - TypeScript 5.5.3
    - Tailwind CSS 3.4.1
    - anime.js 3.2.1 for animations
    - Lucide React 0.344.0 for icons
    - Supabase client 2.57.4
    - ESLint and development tools

    <Tip>
      Installation typically takes 1-2 minutes depending on your internet connection.
    </Tip>
  </Step>

  <Step title="Start the development server">
    Run the Vite development server:

    ```bash
    npm run dev
    ```

    The server will start and display the local URL:

    ```bash
    VITE v5.4.2  ready in 423 ms

    ➜  Local:   http://localhost:5173/
    ➜  Network: use --host to expose
    ➜  press h + enter to show help
    ```

    <Note>
      Vite uses port 5173 by default. If the port is already in use, Vite will automatically select the next available port.
    </Note>
  </Step>

  <Step title="Open in your browser">
    Navigate to the local URL shown in your terminal (typically `http://localhost:5173/`).

    You should see:
    - The DEV WEEK 2025 hero section with animated binary rain
    - Event countdown clock
    - Event details (December 10-12, 2025)
    - Navigation buttons to explore the schedule and speakers
  </Step>
</Steps>

## Verify it's working

Once the site loads, you should see:

1. **Hero section** with the DEV WEEK 2025 logo and 50th anniversary badge
2. **Binary rain animation** falling in the background
3. **Countdown clock** showing time remaining until the event
4. **Event information** displaying dates, time, and location
5. **Navigation buttons** to view the schedule and speakers

<Tip>
  The site uses lazy loading for non-critical components. You may see loading placeholders briefly as you scroll.
</Tip>

## Development workflow

Here's a typical development workflow:

1. **Make changes** to components in `src/components/`
2. **See updates instantly** with Vite's hot module replacement
3. **Check types** with `npm run typecheck` before committing
4. **Lint code** with `npm run lint` to catch errors
5. **Test the build** with `npm run build && npm run preview`

<Warning>
  Always run `npm run typecheck` before pushing changes to ensure type safety.
</Warning>

## Explore the components

The main components are lazy-loaded in `src/App.tsx`:

```tsx src/App.tsx
import { lazy, Suspense } from "react";
import Hero from "./components/Hero";
import Footer from "./components/Footer";

// Lazy load all non-critical components
const About = lazy(() => import("./components/About"));
const Schedule = lazy(() => import("./components/Schedule"));
const Speakers = lazy(() => import("./components/Speakers"));
const LiveStreams = lazy(() => import("./components/LiveStreams"));
const Sponsors = lazy(() => import("./components/Sponsors"));
```

Each component is wrapped in a `<Suspense>` boundary with fallback loading states.
