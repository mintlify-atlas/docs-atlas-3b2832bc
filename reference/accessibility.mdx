# Accessibility

The DEV WEEK 2025 site implements comprehensive accessibility features following WCAG guidelines to ensure the site is usable by everyone, including users with disabilities.

## Skip Links

### Implementation

Skip to main content link allows keyboard and screen reader users to bypass navigation and jump directly to the main content.

**Component Implementation:**
```tsx
// src/App.tsx:15-18
{/* Accessibility: Skip to main content */}
<a href="#main-content" className="skip-link">
  Saltar al contenido principal
</a>

<Hero />

<main id="main-content">
  {/* Content sections */}
</main>
```

**CSS Styling:**
```css
/* src/index.css:96-114 */
/* Accessibility: Skip to main content link */
.skip-link {
  position: absolute;
  top: -40px;
  left: 0;
  background: var(--morado);
  color: var(--blanco);
  padding: 0.75rem 1.5rem;
  text-decoration: none;
  font-weight: 600;
  z-index: 10000;
  transition: top 0.2s ease;
}

.skip-link:focus {
  top: 0;
  outline: 3px solid var(--dorado);
  outline-offset: 2px;
}
```

**Behavior:**
- Hidden by default (positioned off-screen)
- Becomes visible when focused via keyboard (Tab key)
- Clicking jumps to `#main-content`
- High z-index (10000) ensures it's always on top
- High contrast colors (purple background, white text)
- Gold outline on focus for visibility

### Button State (aria-pressed)

**Filter Buttons:**
```tsx
// src/components/Speakers.tsx:137-147
<button
  key={day}
  onClick={() => setSelectedDay(day)}
  className={`${styles.filterBtn} ${
```

**Purpose:**
- Indicates toggle state to screen readers
- `aria-pressed="true"` when filter is active

## Reduced Motion Support

### Media Query

Respects user's motion preferences set at OS level:

```css
/* src/index.css:86-94 */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

**Effect:**
- All CSS animations run for 0.01ms (essentially instant)
- Animation plays once, not looping
- All transitions complete almost instantly
- Applied globally to all elements

### JavaScript Animation Handling

**Motion Detection Function:**
```javascript
// src/utils/animations.js:3-5
const shouldReduceMotion = () => {
  return window.matchMedia('(prefers-reduced-motion: reduce)').matches;
};
```

**Implementation in Animation Functions:**

1. **Early Return Pattern:**
```javascript
// src/utils/animations.js:7-8,20-21
export const enterStagger = (target, delay = 50) => {
  if (shouldReduceMotion()) return; // Skip animation
  // ... animation code
};

export const hoverLift = (target) => {
  if (shouldReduceMotion()) return; // Skip animation
  // ... animation code
};
```

2. **Fallback Animation Pattern:**
```javascript
// src/utils/animations.js:62-70
export const fadeInUp = (target, duration = 800) => {
  if (shouldReduceMotion()) {
    anime({
      targets: target,
      opacity: [0, 1],
      duration: 50,      // Very fast
      easing: 'linear'   // No easing
    });
    return;
  }
  // ... full animation
};
```

**Functions with Reduced Motion Support:**
- `enterStagger()` - Skipped entirely
- `hoverLift()` - Skipped entirely
- `pulseAccent()` - Skipped entirely
- `scaleIn()` - Skipped entirely
- `batchHoverLift()` - Skipped entirely
- `fadeInUp()` - Simplified to opacity fade
- `slideInLeft()` - Simplified to opacity fade

### Color Variables

```css

  --blanco: #ffffff;       /* White */
  --negro: #000000;        /* Black */
  --gris-1: #f5f5f5;       /* Light gray */
}
```

### Text Colors

**Body Text:**
```css
/* src/index.css:40-47 */
body {
  font-family: var(--font-body);
  color: var(--plomo);           /* #535255 - Dark gray */
  background-color: var(--blanco); /* White background */
  line-height: 1.6;
}
```

**Headings:**
```css
/* src/index.css:49-59 */
h1, h2, h3, h4, h5, h6 {
  font-family: var(--font-heading);
  color: var(--negro);  /* Black for maximum contrast */

  font-weight: 700;
}
```

**Contrast Ratios:**
- Body text (#535255 on #ffffff): ~8.7:1 (AAA)
- Headings (#000000 on #ffffff): 21:1 (AAA)
- Purple on white (#61398c on #ffffff): ~6.2:1 (AA)

## Testing Recommendations

### Keyboard Navigation Test
1. Navigate using Tab key only
2. Verify focus indicators are visible
3. Test skip link appears on first Tab press
4. Ensure all interactive elements are reachable
5. Test modal close with Escape key (if implemented)

### Screen Reader Test
1. Test with NVDA (Windows) or VoiceOver (Mac)
2. Verify skip link announces correctly
3. Check ARIA labels are read
4. Confirm button states are announced
5. Test heading hierarchy (H1 → H2 → H3)

### Reduced Motion Test
1. Enable reduced motion in OS settings:
   - Windows: Settings → Ease of Access → Display → Show animations
   - macOS: System Preferences → Accessibility → Display → Reduce motion
2. Reload page and verify animations are minimal
3. Test all interactive elements still work

### Color Contrast Test
Use tools like:
- WebAIM Contrast Checker
- Chrome DevTools Lighthouse
- axe DevTools browser extension

## WCAG Compliance

The site implements:

**Level A:**
- ✅ Text alternatives for images
- ✅ Keyboard accessibility
- ✅ Use of color
- ✅ Focus visible

**Level AA:**
- ✅ Contrast ratio (minimum 4.5:1 for text)
- ✅ Resize text
- ✅ Focus visible
- ✅ Multiple ways to find content

**Level AAA:**
- ✅ Enhanced contrast (7:1 for body text)
- ✅ Animation from interactions
