## Project Organization

The DEV WEEK 2025 project is organized into a clean, modular structure that follows React and TypeScript best practices.

### Root Source Structure

```
~/workspace/source/src/
├── components/        # React components with CSS Modules
├── data/             # Static data and content
├── types/            # TypeScript type definitions
├── utils/            # Utility functions and helpers
├── __tests__/        # Component tests
├── App.tsx           # Main application component
├── main.tsx          # Application entry point
├── index.css         # Global styles and CSS variables
├── animejs.d.ts      # Anime.js type declarations
├── vite-env.d.ts     # Vite environment types
└── types.ts          # Shared type definitions
```

## Key Files

### App.tsx

The main application component that orchestrates the entire page structure:

```typescript
import { lazy, Suspense } from "react";
import Hero from "./components/Hero";
import Footer from "./components/Footer";

// Lazy load all non-critical components for better performance
const About = lazy(() => import("./components/About"));
const Schedule = lazy(() => import("./components/Schedule"));
const Speakers = lazy(() => import("./components/Speakers"));
const LiveStreams = lazy(() => import("./components/LiveStreams"));
const Sponsors = lazy(() => import("./components/Sponsors"));
```

<Info>
  The App component implements **code splitting** with React.lazy() to improve initial load performance by only loading critical components (Hero, Footer) immediately.
</Info>

### main.tsx

The application entry point that renders the React app:

```typescript
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';
import App from './App.tsx';
import './index.css';

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <App />
  </StrictMode>
);
```

### index.css

Global styles, CSS variables, and Tailwind imports. See [Styling](/structure/styling) for details.

## File Naming Conventions

| File Type | Convention | Example |
|-----------|-----------|----------|
| React Components | PascalCase.tsx | `Hero.tsx`, `Schedule.tsx` |
| CSS Modules | PascalCase.module.css | `Hero.module.css` |
| Utility Files | camelCase.ts | `animations.js`, `streamHelpers.ts` |
| Type Definitions | camelCase.ts or .d.ts | `types.ts`, `animations.d.ts` |
| Data Files | camelCase.ts | `schedule.ts` |

<Tip>
  This structure makes it easy to locate files - if you're looking for a component's styles, just look for the matching `.module.css` file in the same directory.
</Tip>
